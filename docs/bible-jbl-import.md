# Импорт перевода JBL (Библия Юбилейная)

## Почему

Старая справочная таблица `bible` (перевод `RST`) оказалась повреждена:
6 книг Нового Завета (1-2 Фес, 1-2 Тим, Титу, Филимону) отсутствовали
полностью, ещё 9 посланий (Рим, 1-2 Кор, Гал, Еф, Флп, Кол, Евр, Иак) были
обрезаны и/или содержали текст не той книги. Затрагивало ~112 из 1820 уже
опубликованных ежедневных записей.

Решение: **не трогать историю**. Старые записи (`task_verse.bible_id`)
как ссылались на старые (RST) строки `bible`, так и ссылаются — без
изменений. Новый перевод добавлен как отдельные строки той же таблицы
(`translation_id = 'JBL'`), и picker в админке (`common\models\Bible::ACTIVE_TRANSLATION_ID`)
переключён на него — все новые/редактируемые записи используют только его.

## Источник

- Файл: `Библия Юбилейная` (исправленный синодальный перевод, с номерами
  Стронга), формат MyBible/SQLite3, `russian_numbering: true` (прямая
  синодальная нумерация глав, без переносной аннотации `(N-M)`, которая
  использовалась в старых RST-данных для Псалмов).
- 66 книг, 31162 стиха.

## Как был сделан импорт

Скрипт: `scratchpad/bible-import/import_jbl.py` (не в репозитории — при
необходимости повторного запуска пересоздать по описанию ниже).

1. Прочитать таблицу `verses` (book_number, chapter, verse, text) и
   `books` (book_number → русское имя) из SQLite.
2. Замапить `book_number` (10, 20, ..., 730 — стандартная MyBible-нумерация)
   на `(book_id, book_name)` в той же английской конвенции, что уже
   использовалась в таблице `bible` (например, `Ps`/`Psalms`,
   `Eph`/`Ephesians`) — для 6 отсутствовавших книг взяты стандартные
   английские названия/аббревиатуры.
3. Очистить текст:
   - `<S>\d+</S>` (номера Стронга) — удалить вместе с содержимым;
   - `<f>...</f>` (маркеры сносок, содержимое — просто номер сноски,
     самого текста сноски в дампе нет) — удалить вместе с содержимым;
   - `<br/>`, `<pb/>` — заменить на пробел;
   - `<i>`, `<e>`, `<t>`, `<J>` (и закрывающие) — снять тег, текст внутри
     оставить;
   - схлопнуть `..` (артефакт после удаления тегов) в `.`, не трогая
     настоящее троеточие `...`;
   - схлопнуть повторные пробелы.
4. Сгенерировать `INSERT INTO bible (...) VALUES (...)` пачками по 500
   строк, с предварительным `DELETE FROM bible WHERE translation_id='JBL'`
   (идемпотентность при повторном запуске).
5. Применить сгенерированный `.sql` к базе (`mysql ... < jbl_import.sql`).

## Известные мелочи источника (оставлены как есть)

- `«Мир... и прочее»` (Ездра 4:17) — настоящее многоточие в оригинале,
  не артефакт.
- Дефис/тире в тексте — уже используется корректное «–» (в отличие от
  старых RST-данных, где встречается `--`).

## Проверено

- 66 книг / 31162 стиха после импорта — совпадает с источником.
- Ефесянам, 1 Фессалоникийцам и другие ранее битые/отсутствующие книги —
  корректное количество глав и текст.
- Псалом 108 — прямая нумерация, без `(107-1)`-аннотаций, тире `–` вместо `--`.
- Существующие (RST) записи `task/find` по-прежнему возвращают исходный
  текст без изменений.
